NGINX

发表于 2021-08-05 分类于 Java EE

NGINX 简介

NGINX：engine x 的简写

主要功能：

  • 负载均衡
  • 反向代理
  • 动静分离

Linux 中安装 NGINX

  1. 下载 NGINX：http://nginx.org/en/download.html

  2. 把下载的压缩包 nginx-1.21.1.tar.gz 放到 /usr/local/ 目录下，然后解压

  3. 进到解压后的 nginx-1.21.1 目录中，执行命令 ./configure --prefix=/usr/local/nginx-1.21.1

  4. 执行 make 命令，然后执行 nake install 命令，就安装完成了

进入 nginx-1.21.1/sbin 目录，执行 ./nginx 开启服务

开启服务的时候遇到一个保存

是因为在 nginx-1.21.1 目录下没有 logs 目录，新建一个就好了

NGINX 使用

基本命令：

  • nginx：开启服务
  • nginx -s reload：重新加载配置文件
  • nginx -t：检查配置文件是否有错误
  • nginx -s reopen：重启服务
  • nginx -s quit：停止服务
  • nginx -s stop：强制停止服务

NGINX 配置文件：

位于 nginx-1.21.1/conf/nginx.conf

使用 NGINX 主要就是修改配置文件中的各种配置，从而实现不同的功能

Stream

Stream 是 Java 8 提供的一种以函数式的方式处理集合的 API

像 SQL 一样操作 Java 集合中的数据

使用 Stream 的三个步骤：

  1. 创建流
  2. 中间操作
  3. 终止操作

如果没有终止操作，那么所有的中间操作都不会执行

创建流

三种创建流的方式：

  1. 通过 Collection 接口提供的默认方法： stream() 或 parallelStream()
    • Stream()：创建串行流
    • parallelStream()：创建并行流
  2. 通过 Arrays 工具类提供的静态方法：stream()
  3. 通过 Stream 类中的静态方法 of()

中间操作

  1. 筛选和切片

    filter()：过滤（接收一个 Lambda）

    distinct()：去重

    limit()：保留前几个元素

    skip()：排除前几个元素

  2. 映射

    map()：传入一个 Function，这个 Function 会作用于集合中所有的元素

    flatMap()：如果一个 Function 返回的是一个流，flatMap 会把返回的所有流合并成一个流

    map 和 flatMap 的关系类似于 List 中的 add 和 addAll

  3. 排序

    sort()：自然排序

    sort(Comparator)：传入一个比较器实例，自定义排序方式

终止操作

查找与匹配：

  • allMatch()：是否匹配全部元素
  • anyMatch()：是否匹配任意元素
  • noneMatch()：是否没有元素匹配
  • findFirst()：返回第一个
  • findAny()：返回任意一个

归约：

  • reduce()：按照一定的规则对 Stream 中的值进行操作，最终得到一个最终结果

收集：

  • collect()：将流转换成集合（List / Set / Map）

forEach()：内部迭代，遍历集合

count()：统计集合中元素的数量

max()

min()
