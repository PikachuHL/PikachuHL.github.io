第二章：简单的 HTTP 协议

在两台计算机之间使用 HTTP 协议通信时，在一条通信线路上必定有一端是客户端，另一端是服务器端。

请求报文是由：请求方法、请求 URI、协议版本、可选的请求首部字段、内容实体构成

响应报文是由：协议版本、状态码、原因短语、可选的响应首部字段、实体主体构成

HTTP 是一种无状态的协议，即：在 HTTP 这个级别，协议对发送过的请求或响应都不做持久化处理。这样设计是为了更快的处理大量事务，确保协议的可伸缩性。

Profile

一个项目，如果希望在不同的环境下加载不同的 bean，需要两步：

  1. 将不同的 bean 放到一个或多个 profile 中
  2. 将应用部署到不同环境时，确保对应的 profile 处于激活状态

被激活的 profile 中的 bean 会被创建，未被激活的 profile 中的 bean 不会被创建，没有放到任何 profile 中的 bean 总会被创建

配置 profile

使用 @Profile 注解指定某个 bean 属于哪一个 profile

@Profile 注解可以用在配置类上，一次性将多个 bean 放到一个 profile 中

@Profile 注解也可以用在 @Bean 标注的方法上，一次将一个 bean 放到一个 profile 中

@Profile 注解也可以用在 @Component 注解标注的类上，一次将一个 bean 放到一个 profile 中

在 XML 配置文件中，使用 <beans> 元素的 profile 属性指明该配置文件中的 bean 属于哪一个 profile

<beans> 元素内部可以嵌套定义 <beans> 元素，所以可以在一个 XML 文件中配置多个 profile

激活 profile

Spring 通过 spring.profiles.active 和 spring.profiles.default 两个属性的值来确定激活那个 profile

  • 如果 active 有值：激活 active 指定的 profile
  • 如果 active 没值，default 有值：激活 default 指定的 profile
  • 如果 active 没值，default 也没值：没有 profile 被激活，只创建未定义在 profile 中的 bean

如何设置 active 和 default 属性的值：

  • 作为 DispatcherServlet 的初始化参数
  • 作为 Web 应用的上下文参数
  • 作为 JNDI 条目
  • 作为环境变量
  • 作为 JVM 的系统属性
  • 在集成测试类上，使用 @ActiveProfiles 注解设置

条件化 bean

如果希望一个 bean 在某个条件满足时才创建，可以使用 @Conditional 注解（Spring 4 开始提供）

@Conditional 注解可以用在 @Bean 修饰的方法上，传入一个类的 Class，这个类实现了 Condition 接口，提供了接口中 matches 方法的实现，当该方法返回 true 时，就会创建这个 bean，返回 false 就不会创建

  1. AnnotatedTypeMetadata：也是一个接口

从 Spring 4 开始，@Profile 注解基于 @Conditional 注解和 Condition 接口进行了重构

@Autowired 注解只有当容器中有且只有一个要注入类型的 bean 时才可以注入成功，如果同类型的 bean 有多个，@

Autowired 就无法注入了

标示首选的 bean

在 JavaConfig 中，使用 @Primary 注解标注哪个为首选 bean

在 XML 配置文件中，使用 <bean> 标签的 primary 属性标示首选 bean

首选 bean 只能有一个，如果多个同类型的 bean 都被标注为首选 bean，也就不存在首选 bean 了

@Primary 注解只能指定一个首选 bean 用来自动注入，如果想要注入指定的 bean，就需要 @Autowired 搭配 @Qualifier 注解使用

Spring In Action/第2章-装配Bean

发表于 2021-08-24 更新于 2021-08-26

Spring 配置 bean 有三种方案：

  • 在 XML 中进行显式配置
  • 在 Java 中进行显式配置
  • 隐式的 bean 发现机制和自动装配

自动装配 bean

Spring 从两个角度来实现自动装配：

  • 组件扫描（Component Scanning）：Spring 会自动发现应用上下文中所创建的 bean
  • 自动装配（Autowiring）：Spring 自动满足 bean 之间的依赖关系

组件扫描

在类上添加 @Component 注解，表示把该类作为 Spring 的一个组件类，并告诉 Spring 需要为这个类创建 bean

注意：Spring 的组件扫描默认是不开启的，需要显示配置组件扫描，才能让 Spring 找到带有 @Component 注解的类， 并为其创建bean

Spring 容器中的每个 bean 都有一个 ID，如果仅使用 @Component 注解修饰类，Spring 会把类名首字母变小写作为 bean 的 ID。

如果想要给 bean 自定义 ID，使用 @Component("customBeanName") 即可

使用 @ComponentScan 注解开启组件扫描，该注解需要写在配置类（有 @Configuration 注解的类）上

扫描规则：

  • 默认扫描配置类所在包及其子包

  • 添加属性指定要扫描的包

    • @ComponentScan(basePackages = {"cn.hellopika.domain"})：指定要扫描的包名（可一次指定多个）

    • @ComponentScan(basePackageClasses = {CDPlayer.class})：指定类的 Class，扫描类所在包（可一次指定多个）

      最佳实践：在要扫描的包中添加一个空接口（EmptyInterface），@ComponentScan(basePackageClasses = {EmptyInterface.class})

      这样就不用担心后期维护中相关类出现问题（传入 @ComponentScan 的类可能会被移除）

自动装配

使用 @Autowired 注解实现自动装配，该注解可以作用在属性、构造方法、普通方法、参数上

@Autowired 注解作用在属性上，会在容器中自动寻找与属性类型相同的 bean 注入进来

@Autowired 注解作用在方法上，会在容器中自动寻找与参数类型相同的 bean 注入进来

@Autowired 注解只能按照类型注入，当容器中有且仅有一个 bean 的类型与要注入的类型一致时，才可成功注入

@Autowired 注解只有一个属性：required，默认为 true，表示这个 bean 是必须的。如果想设为 false，需要注意在业务逻辑中对这个 bean 进行判空，否则可能会出现 NullPointerException

在 Java 中显式配置 bean

上面，我们使用了 @Component 注解把一个类作为 Spring 的组件类，扫描到这个类之后就在容器中创建 bean。但是，如果想把一些第三方的类交给 Spring 管理，我们就无法为类添加 @Component 注解，需要使用 @Bean 注解显式配置 bean

@Bean 注解可以在 @Configuration 注解或 @Component 注解修饰的类中使用

@Bean 注解作用在方法上，会把方法的返回值作为 bean 放到 Spring 容器中，默认以方法名作为 bean 的 ID

可以通过 name 属性自定义 ID：@Bean(name = "customBeanName")

在 XML 中装配 bean

Spring 的 XML 文件的配置规范：

声明 bean：

<bean class="cn.hellopika.domain.CDPlayer" />

通过 <bean> 标签的 class 属性声明，class 中使用类的全限定类名，在容器中 bean 的 ID 默认为 全限定类名#0，比如刚才定义的这个 bean 的 ID 为 cn.hellopika.domain.CDPlayer#0，其中的 #0 为计数，如果容器中有多个同类型的 bean 时，# 后面的数字会递增

在实际使用中，最好自己通过 id 属性自定义 bean 的 ID：

<bean id="cdPlayer" class="cn.hellopika.domain.CDPlayer" />

上述方式声明的类，Spring 会调用默认的构造方法在容器中创建 bean

构造器注入

如果类的构造方法被重写，而且有参数，此时就涉及到构造器注入

构造器注入有两种基本的配置方案：

  • <constructor-arg>
  • c-命名空间（Spring 3.0 引入的）

使用 <constructor-arg>：

使用 c-命名空间

使用 c-命名空间需要在 XML 头部添加模式声明：xmlns:c="http://www.springframework.org/schema/c"

c:cd-ref="compactDisc" 属性格式解析：

  • c:：命名空间前缀
  • cd：构造器参数名
  • -ref：告诉 Spring，正在装配的是一个 bean 的引用
  • "compactDisc"：引用的 bean 的 ID

属性的其他写法：（上面那种写法并不常用）

  • c:_0-ref="compactDisc"：使用参数索引而不是参数名称（有多个参数时比较好用）
  • c:_-ref="compactDisc"：构造器中只有一个参数时的简写形式

构造器参数为字面量时，属性就不加 -ref，像这样：

  • c:_name="hello"：使用参数名称
  • c:_0="hello"：使用参数索引
  • c:_="hello"：只有一个参数

上面说了构造器参数为其他 bean 和字面量两种情况，使用 <constructor-arg> 和 c-命名空间 都是可以的。但是，如果参数为集合，就只能使用 <constructor-arg> 注入

Set 方法注入

使用 Set 方法注入也有两种配置方案：

  • <property>
  • p-命名空间

使用 <property>

使用 p-命名空间

使用 p-命名空间需要在 XML 头部添加模式声明：xmlns:p="http://www.springframework.org/schema/p"

和构造器注入一样，set 方法注入时，集合只能使用 <property> 注入

导入配置

当 JavaConfig 中需要配置的 bean 非常多时，我们会考虑将不同类别的 bean 放到不同的 JavaConfig 中，从而方便我们管理

在 Spring 中，我们可以同时使用 JavaConfig 和 XML 配置文件

在 JavaConfig 中导入其他配置

  • 在 JavaConfig 中导入其他 JavaConfig：使用 @Import 注解

  • 在 JavaConfig 中导入 XML 配置文件：使用 @ImportResource 注解

在 XML 配置文件中导入其他配置

  • 在 XML 配置文件中导入其他 XML 配置文件：使用 <import> 标签

  • 在 XML 配置文件中导入 JavaConfig：使用 <bean> 标签
