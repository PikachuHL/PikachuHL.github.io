注解

发表于 2019-03-28 更新于 2021-05-12

注解和注释有相似之处，但也有比较大的差别

注解 - Annotation

注解本质上就是一个接口，该接口默认继承 Annotation 接口

1、注解的作用

  • 不是程序本身，可以对程序进行解释
  • 可以被其他程序读取

2、注解的格式

@注解名

3、注解的使用范围
package、class、method、field

4、内置注解

  • @Override –> 该注解只适用于修饰方法，表示该方法重写了父类(或接口)的方法
  • @Deprecated –> 该注解可用于修饰方法、属性、类，表示不推荐使用这样的元素
  • @SuppressWarnings –> 用来抑制编译时的警告信息

自定义注解

定义关键字：@interface

属性：注解中的抽象方法

  • 由于注解是接口，所以注解内可以定义 抽象方法

    在注解内，抽象方法又被称为属性

  • 抽象方法（属性）的返回值类型仅可为以下几种：

    • 基本数据类型
    • String
    • 枚举
    • 注解
    • 以上类型的数组

  • 定义了属性，在使用时需要对属性进行赋值

    • 如果定义属性时，使用 default 关键字给属性默认初始化值，则使用注解时，可以不进行属性的赋值
    • 如果只有一个属性需要赋值，并且属性的名称是 value，则 value 可以省略，直接定义值即可
    • 数组赋值时，值使用 {} 包裹。如果数组中只有一个值，则 {} 可以省略

元注解：

  • @Target：表示注解可以修饰的范围

    • PACKAGE –> 包
    • TYPE –> 类、接口、枚举、注解
    • CONSTRUCTOR –> 构造器
    • FIELD –> 域
    • METHOD –> 方法
    • LOCAL_VARIABLE –> 局部变量
    • PRARMETER –> 参数

  • @Retention：描述注解的生命周期

    • SOURCE –> 在源文件中有效
    • CLASS –> 在 class 文件中有效
    • RUNTIME –> 在运行时有效

  • @Documented：描述注解是否被抽取到 API 文档中

  • @Inherited：描述注解是否被子类继承

使用反射机制读取注解

只有将注解读取出来，注解才有实际的使用意义。
