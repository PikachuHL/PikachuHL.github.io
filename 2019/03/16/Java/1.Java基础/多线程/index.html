多线程

发表于 2019-03-16 更新于 2021-04-16 分类于 Java

多线程在 Java 中多应用于网络编程和服务器端应用开发

实现多线程的方法

1.继承 Thread 类

2.实现 Runnable 接口

优先使用接口实现多线程，可避免单继承带来的局限性

3.实现 Callable 接口

Callable 接口比较高级，涉及到高并发的时候回比较常用。

实现 Callable 接口需要重写 call() 方法，该方法可以有返回值类型，也可以抛出异常

线程状态

  1. 新生状态 –> New

    new 对象，就进入新生状态

  2. 就绪状态 –> RUNNABLE
    以下四种情况会使线程进入就绪状态

    • start()
    • 阻塞状态解除
    • yield()
  • JVM 干预
  1. 运行状态 –> RUNNABLE

cpu 调度到了，就进入运行状态

  1. 阻塞状态 –> Waiting / Timed_Waiting / Blocked

    以下四种情况会使线程进入阻塞状态

    • sleep() –> WAITING/TIMED_WAITING
    • join() –> WAITING/TIMED_WAITING
    • wait() –> BLOCKED
    • read,write –> BLOCKED

  2. 死亡状态 –> TERMINATED

    不推荐使用 stop() 和 destory() 来停止线程，因为这两个存在 bug

    可以使用添加 boolean 标志的方法停止线程，方便外部进行控制

暂停 sleep

  • sleep 为静态方法，可以直接在线程中调用
  • sleep (毫秒数)指定当前线程阻塞的时间
  • sleep 存在异常 InterruptedException
  • sleep 时间达到后线程进入就绪状态
  • sleep 可以模拟网络延时、倒计时等
  • 每一个对象都有一个锁，sleep 不会释放锁（抱着对象睡觉）

礼让 yield

  • yield 也是静态方法，可以在线程中直接调用
  • 礼让线程，让当前正在执行的线程从运行状态转入就绪状态
  • 让 cpu 调度器重新调度

插队 join

使用 join 后，需要等插队的线程执行完成后才能执行其他线程，其他线程阻塞

优先级

  • 优先级的取值范围在 1-10 之间
  • 优先级有三个常量
    • MAX_PRIORITY –> 10
    • MIN_PRIORITY –> 1
    • NORM_PRIORITY –> 5
  • 在 start() 之前设置优先级
  • 优先级高的不一定先执行，优先级只是代表执行的概率

守护线程

  • 所有的线程默认都是用户线程，守护线程需要手动设置
  • JVM 停止不用考虑守护线程

设置守护线程
t.setDaemon(true)

线程同步

1.概念：队列+锁=线程安全=线程同步

2.锁机制（synchronized）

  • 锁机制保证了线程的安全，但是带来了性能的问题

  • 用法：
    synchronized 方法 –> 同步方法
    synchronized 块 –> 同步块

    使用 synchronized 要注意：

    - 锁准对象，否则不能保证线程安全
- 锁准范围，如果将一个大的方法声明为 synchronized，将会大大影响效率

    同步块：synchronized（obj）{ } ， obj 称之为同步监视器
    obj 就是被锁的对象，就是要修改的那个对象
    多用同步块，少用同步方法，因为同步块在保证线程安全的前提下，性能更优

  • 死锁
    过多的同步可能造成相互不释放资源，从而相互等待，一般发生于同步中持有多个对象的锁。
    避免死锁：不要在同一个代码块中同时持有多个对象的锁
