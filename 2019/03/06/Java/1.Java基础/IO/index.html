Java.IO基本概念及File类

Java.IO

Java.IO包中最重要的就是5个类和三个接口

  • File 文件类
  • InputStream 字节输入流
  • OutputStream 字节输出流
  • Reader 字符输入流
  • Writer 字符输出流
  • Closeable 关闭流接口
  • Flushable 刷新流接口
  • Serializable 序列化接口

流的分类

按方向分：

  • 输入流
  • 输出流

按功能分：

  • 节点流：直接操作数据源，所有的IO操作必须经过节点流进行
  • 处理流（包装流）：为了简化操作和提高性能

按数据分：

  • 字节流
  • 字符流（底层还是基于字节流操作，自动搜寻了指定的码表）

API的使用

  • 再看构造器（有/无）
  • 最后看方法（方法名、接收参数、返回类型、调用者）

File类的常用方法：

1.基本信息：

  • getName -> 获得文件名
  • getPath -> 获得传入的路径
  • getAbsolutePath -> 获得绝对路径
  • getParent -> 获得父路径

2.文件状态：

  • exists -> 是否存在
  • isFile -> 是否是文件
  • isDirectory -> 是否是文件夹

3.文件长度：

  • length() -> 返回文件的长度（字节数），文件夹的length()返回0

4.创建&删除文件：

  • createNewFile() -> 不存在创建成功，已存在创建失败
  • delete()

5.创建目录（文件夹）：

  • mkdir() -> 如果上级目录不存在，则会创建失败
  • mkdirs() -> 如果上级目录不存在，则一同创建

6.列出下级目录：

  • list() -> 列出下级文件名，返回一个数组
  • listFiles() -> 列出下级文件对象，返回一个数组
  • listRoots() -> 列出所有根目录，返回一个数组

文件字节流

  • FileInputStream

    文件字节输入流，通过字节方式读取文件
    全字符请考虑：FileReader

  • FileOutputStream

    文件字节输出流，通过字节的方式写出或追加数据到文件
    全字符请考虑：FileWriter

文件拷贝

利用FileInputStream和FileOutputStream进行文件的拷贝

IO流其实就是一个管子，把字节或字符放到管子里面进行操作

文件字符流

文件字符流可以消除不同字符集的编码差异

  • FileReader
  • FileWriter
    FileWriter中有一个需要注意的方法：
    append() // 返回值类型为Writer，可以以‘流模式’来添加

文件字符流操作的是字符数组
文件字节流操作的是字节数组

字节数组流

  • ByteArrayInputStream
  • ByteArrayOutputStream

字节数组流操作的是内存中的字节数组，不与OS打交道，故可由gc自动回收，不用关闭
任何东西都可以转成字节数组，可以方便后期进行网络传输等操作
