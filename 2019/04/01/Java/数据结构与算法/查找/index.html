查找

发表于 2019-04-01 更新于 2020-05-26

查找和排序是数据结构中两个最重要的操作

查找

一、线性表查找

1、顺序查找

非常简单

查找方法：逐个比较查询，如果找到，返回数据或者索引，如果没找到，返回null

时间复杂度：T(n) = O(n)

2、折半（二分）查找

查找表进行折半查找必须满足两个条件：

  • 查找表必须使用顺序存储结构
  • 查找表必须按关键字大小有序排列

折半查找可以使用两种方式来实现：

  • 非递归方式

    时间复杂度：T(n) = O(log2n)

    空间复杂度：S(n) = O(1)

  • 递归方式

    时间复杂度：T(n) = O(log2n)

    空间复杂度：S(n) = O(log2n)

二、查找树

二叉查找树 BST（Binary Search Tree）：

或是一棵空树，

或是具有如下性质的二叉树：

  • 若它的左子树不为空，则左子树上的所有节点的值小于根节点的值
  • 若它的右子树不为空，则右子树上的所有节点的值大于根节点的值
  • 它的左、右子树也都为二叉查找树

对二叉查找树进行中序遍历，得到有序集合

平衡二叉树（self-balance Binary Search Tree）:自平衡二叉查找树，又称AVL树

或是一棵空树

或它的左、右子树的高度差（平衡因子（平衡度））的绝对值不大于1

意义：减少二叉查找树层次，提高查找速度

红黑树（R-B Tree）：是一种平衡二叉树

特性：

  • 每个节点都有颜色，或是黑色，或是红色
  • 根节点是黑色
  • 每个叶子节点（null）是黑色
  • 如果一个节点是红色的，那么它的子节点一定是黑色的
  • 从一个节点到该节点的子孙节点的所有路径上包含相同数目的黑节点

应用：

Java集合中的TreeSet和TreeMap就是使用红黑二叉树实现的

B树（Balance Tree）: 平衡树

平衡树可以降低树的深度，提高查找效率

B树应文件系统的要求而发展起来的

B+树：

在B树的基础上，为叶子节点增加链表指针，所有关键字都在叶子节点中出现，非叶子节点只作为索引

数据库的索引的默认结构采用的就是B+树

B*树：

B+树的变体，在非根和非叶子节点再增加执行兄弟的指针
