在 CentOS 上安装 Oracle 数据库的步骤

在 Linux 上安装 Oracle

Linux 版本：CentOS 7

Oracle 版本：Oracle-XE-11.2.0

Oracle 下载地址：https://www.oracle.com/database/technologies/xe-prior-releases.html

参考安装步骤：https://docs.oracle.com/cd/E17781_01/install.112/e18802/toc.htm#XEINL102

下载

打开下载页面，下载 for Linux 的 Oracle，如下图：

下载下来的是一个压缩包 oracle-xe-11.2.0-1.0.x86_64.rpm.zip，解压后找到 oracle-xe-11.2.0-1.0.x86_64.rpm 文件，使用 Xftp 拷贝到 Linux 中

使用压缩包中的 xe.rsp 文件，可以执行静默安装（本次安装采用的仍然是常规安装方式）

安装

  1. 使用 root 用户登录到 Linux

  2. 使用 rpm -ivh oracle-xe-11.2.0-1.0.x86_64.rpm 安装 Oracle

    安装前要确保有足够的 swap 内存，swap 内存不足就无法安装 Oracle

  3. 安装过程中提示需要配置 Oracle 时，执行 /etc/init.d/oracle-xe configure 进行配置：

  4. 配置完成后会自动完成安装

服务

开启服务：/etc/init.d/oracle-xe start

关闭服务：/etc/init.d/oracle-xe stop

环境变量

不需要手动配置，直接运行 Oracle 提供的文件就可以配置基本的环境变量

  1. cd 到 /u01/app/oracle/product/11.2.0/xe/bin
  2. source oracle_env.sh

上面两步可以为本次登录的当前 shell 设置环境变量，如果想要持久的修改环境变量，需要修改配置文件：

远程连接

默认情况下 Oracle 只支持本机连接，如果想要远程连接：

  1. 先打开 SQL 命令行： sqlplus

  2. 然后使用 system 用户登录

  3. 登录成功后设置：EXEC DBMS_XDB.SETLISTENERLOCALACCESS(FALSE);

Oracle的默认端口为1521，需要打开 Linux 防火墙的 1521 端口

远程连接时遇到的问题：

防火墙 1521 端口打开了，也设置 Oracle 支持远程连接了，但是远程 Navicat 中还是报连接超时的错误。

解决：关闭 Linux 的防火墙，就可以连接成功了

疑惑：第一次连接成功后，就算再把 Linux 的防火墙打开，还是可以连接成功，难道只有第一次连接需要关闭防火墙？？？
