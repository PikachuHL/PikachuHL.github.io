Java8新特性

发表于 2021-08-10 分类于 Java 阅读次数：

Java 8 提供的新特性，今后开发会常用

Lambda 表达式

Lambda 表达式是 Java 8 的新特性，为了简化类（只使用一次的类）的书写。

接口中只有一个方法时才能使用 Lambda 表达式

当接口中的方法有参数时：

四个内置的函数式接口

  1. Consumer<T>：消费型接口

    void accept(T t);

  2. Supplier<T>：供给型接口

    T get();

  3. Function<T, R>：函数型接口

    R apply(T t);

  4. Predicate<T>：断言型接口

    boolean test(T t)

方法引用

如果 Lambda 体中的内容已经有方法实现了，可以直接使用方法引用简化 Lambda 表达式

普通方法引用：对象名::方法名 or 类名::方法名

静态方法引用：类名::静态方法名

构造方法引用：类名::new

使用方法引用的注意事项：

  1. Lambda 体中调用方法的参数列表和返回值类型 与 函数式接口中抽象方法的参数列表和返回值类型一致时，才能使用方法引用。
  2. 若 Lambda 参数列表的第一个参数是方法的调用者，第二个参数是方法的参数时，才可以使用 类名::方法名 的形式
  3. 构造方法引用时，构造器的参数列表与函数式接口中抽象方法的参数列表要保持一致

Stream

Stream 是 Java 8 提供的一种以函数式的方式处理集合的 API

像 SQL 一样操作 Java 集合中的数据

使用 Stream 的三个步骤：

  1. 创建流
  2. 中间操作
  3. 终止操作

如果没有终止操作，那么所有的中间操作都不会执行

创建流

三种创建流的方式：

  1. 通过 Collection 接口提供的默认方法： stream() 或 parallelStream()
    • Stream()：创建串行流
    • parallelStream()：创建并行流
  2. 通过 Arrays 工具类提供的静态方法：stream()
  3. 通过 Stream 类中的静态方法 of()

中间操作

  1. 筛选和切片

    filter()：过滤（接收一个 Lambda）

    distinct()：去重

    limit()：保留前几个元素

    skip()：排除前几个元素

  2. 映射

    map()：传入一个 Function，这个 Function 会作用于集合中所有的元素

    flatMap()：如果一个 Function 返回的是一个流，flatMap 会把返回的所有流合并成一个流

    map 和 flatMap 的关系类似于 List 中的 add 和 addAll

  3. 排序

    sort()：自然排序

    sort(Comparator)：传入一个比较器实例，自定义排序方式

终止操作

  1. 匹配：

    allMatch()：是否匹配全部元素

    anyMatch()：是否匹配任意元素

    noneMatch()：是否没有元素匹配

  2. 查找：

    findFirst()：返回第一个

    findAny()：返回任意一个

    count()：返回元素的数量

    max()：返回最大值

    min()：返回最小值

  3. 归约：

    reduce()：按照一定的规则对 Stream 中的值进行操作，最终得到一个最终结果

  4. 收集：

    collect()：汇总流中的元素，传入一个 Collector 接口的实现类（一般是 Collectors）

    Collectors 常用方法：

    • toList()：把流中元素转化为 List
    • toSet()：把流中元素转化为 Set
    • summingDouble()：总和
    • averagingDouble()：平均值

  5. 遍历

    forEach()：内部迭代，遍历整个流
